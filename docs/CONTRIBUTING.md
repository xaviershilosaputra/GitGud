# Contributing to GitGud

First of all, thank you for being interested in helping with GitGud. This project is built to help people learn Git through a story and any kind of help is welcome. Whether you are a writer or a coder, your input makes the project better.

## How you can help

There are many ways you can contribute to this project:

* **Story and Narrative:** You can help me build the lore of the Kethara Repository. If you have ideas for the dialogue or the plot of the five arcs, feel free to suggest them.
* **Questions and Tasks:** The learning part of the app can always be better. You can refine the existing Git tasks or add new unique question types that challenge the user in different ways.
* **Bug Fixes:** If you find a bug in the terminal simulation or the logic puzzles, please let me know or submit a fix for it.
* **UI and UX:** I want the site to look great and feel smooth. You can help by decorating the styling, improving the layout, or making the interface more professional.
* **Accessibility:** Making the site accessible for everyone is very important. Help me make GitGud WCAG compliant so everyone can learn version control.
* **Multiplayer Support:** One of the big future goals is a multiplayer race mode. If you have ideas or code to help make this happen, that would be a huge help.
* **General Improvements:** Anything that makes the site better overall is a valid contribution.

## The process

1. Fork the repository to your own GitHub account.
2. Create a branch for the specific feature or fix you are working on.
3. Make your changes in the code.
4. Push the changes to your fork.
5. Open a pull request with a clear description of what you did and why.

## Style and Code

Please try to keep the code clean and follow the existing structure in the folders. For the CSS and JS, make sure your changes do not break the responsive design on mobile or desktop.

## Feedback

If you have an idea but you are not sure how to code it yet, you can still open an issue to discuss it. I am happy to hear your thoughts on how to make GitGud the best way to learn Git.
